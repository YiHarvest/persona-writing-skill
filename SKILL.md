# 人物文风工厂

把人物卡管理和文风改写作为同一套闭环执行。所有相对路径均以本 Skill 根目录为基准。

## 路由请求

按步骤判断用户意图并只读取完成该意图所需的模块：

1. 判断用户意图（例如：匹配或创建人物卡 / 指定人物卡改写 / 使用当前人物卡改写 / 未指定人物卡改写 / 处理不满意反馈）。
2. 根据意图只读取必要的参考与卡片文件，避免多余模块加载。
3. 按优先级选择人物卡（指定 > 当前 > 历史专属 > 种子），除非用户显式覆盖优先级。
4. 执行该意图对应的流程步骤（立骨、铸魂、画皮、动嘴等），并在最终输出前读取 `references/15-最终输出规范.md` 完成格式与标签。

具体分流：

1. **匹配或创建人物卡**：读取 `references/01-用户画像提取.md`、`02-人物卡匹配.md`；需要生成时再读取 `03-人物卡生成.md`、`04-人物卡模板.md`、`05-人物卡质量检查.md`。
2. **指定人物卡改写**：读取指定卡、`references/10-立骨.md`、`12-画皮.md`、`13-动嘴.md`、`15-最终输出规范.md`；跳过铸魂。
3. **使用当前人物卡改写**：读取 `records/当前人物卡.md` 指向的卡，再执行立骨、画皮、动嘴，并按 `references/15-最终输出规范.md` 输出；跳过铸魂。
4. **未指定人物卡改写**：在未指定人物卡的情况下，自动选择最匹配的卡进行改写。读取 `references/10-立骨.md`、`11-铸魂.md`，动态扫描 `cards/custom/*.md` 和 `cards/seeds/*.md`，选卡后执行画皮、动嘴，并按 `references/15-最终输出规范.md` 输出。
5. **处理不满意反馈**：读取 `references/14-文风改写主流程.md` 的反馈分流。执行问题保留卡片重做画皮和动嘴；卡片问题生成专属卡、保存、设为当前卡，并立即重写上一版原文；两类都按 `references/15-最终输出规范.md` 输出。

## 执行约束

- 按 `当前卡 > 历史专属卡 > 种子卡` 的顺序优先匹配；用户本轮明确指定时，以指定卡为最高优先。
- 只使用磁盘中实际存在的人物卡，不凭记忆补人物名，不限制文件名必须以数字开头。
- 默认在内部完整执行分析，只向用户展示简要过程和最终稿；用户要求时再展示完整拆解或候选评分。
- 每一种改写模式都必须在写最终回复前读取 `references/15-最终输出规范.md`，不能凭通用习惯自行省略标题、标签或已执行步骤的完成标记。
- 保留原文的事实、核心观点、关键数据和必要限定；不得虚构事实、数据、引语、来源、个人经历或身份。
- 模仿人物卡的抽象特征，包括语气、句式、节奏、结构和修辞规律；不得机械复制卡片中的经典金句、口头禅或高度可识别原句。
- 将卡片中的个人履历和第一人称经历视为风格研究材料，不移植为用户经历。除非用户明确提供，不让成稿冒充卡片人物本人。
- 涉及新闻、政策、价格、人物职位、科技进展、医学、法律、金融等时效或高风险事实时，先核验再补充；无法核验就只改写用户已提供的信息并保留不确定性。
- 用户只要求改写时直接完成，不因人物卡确认而中断；自动选择最佳卡并在结果中简要说明。
- 除非用户明确要求“只要成稿”或“不要解释”，自动选卡的最终回复必须包含 `✅ 铸魂完成：选定「人物卡」— 一句理由`；不得只在内部选卡而省略这一行。
- 写入专属卡或记录前，遵循 `references/03-人物卡生成.md` 的命名与更新规则，避免覆盖现有文件。

## 资源导航

- `cards/seeds/`：8 张初始人物卡。
- `cards/custom/`：经用户确认或明确要求保存的专属人物卡。
- `records/当前人物卡.md`：当前人物卡指针，不复制整张卡。
- `records/人物卡索引.md`：所有卡片的索引；新增卡后同步更新。
- `records/用户反馈记录.md`：只记录与风格优化直接相关的非敏感反馈。
- `references/14-文风改写主流程.md`：完整编排和异常处理。
