# 人物卡生成与保存

## 生成条件

在以下任一情况生成专属人物卡：

- 用户明确要求创建专属卡。
- 用户确认已有卡均不合适。
- 反馈表明人设、领域、受众、价值立场或整体表达气质不匹配，而非单次执行偏差。

只有“太长、开头太夸张、数据太少、某句太激烈”等局部问题时，不新建卡；保留当前卡并重做画皮和动嘴。

## 生成步骤

1. 读取 `01-用户画像提取.md` 并形成有证据的创作画像。
2. 读取当前卡及用户明确指出的不匹配点，提炼保留项、删除项和新增项。
3. 按 `04-人物卡模板.md` 生成完整卡片。
4. 按 `05-人物卡质量检查.md` 自检并修正。
5. 先在对话中给出卡片摘要；用户明确要求保存，或当前请求已明确包含“生成并保存”时，再按 `00-状态管理.md` 调用状态脚本。

## 保存规则

- 使用 `uv run --script scripts/persona_state.py save` 保存，不直接编辑卡库、索引或当前指针。
- 默认保存目录：`~/.persona-writing-skill/cards/custom/`；可由 `PERSONA_WRITING_HOME` 或 `--state-dir` 覆盖。
- 文件名：`YYYYMMDD-卡片名.md`；同名已存在时追加 `-02`、`-03`，不得覆盖。
- 卡片名使用简短、可识别、无路径符号的中文或字母数字名称。
- 保存时传入领域和核心风格；脚本会同步索引。需要设为当前卡时传入 `--set-current`。
- 使用 `add-feedback` 追加日期、原卡、问题类型、关键反馈和处理结果；不记录无关隐私。
- 若生成源于改写不满意，保存并设为当前卡后，立即使用新卡重新执行上一原文的画皮和动嘴，不要求用户重发原文。

## 更新而非新增

用户明确说“修改这张专属卡”时，重新调用 `save --set-current` 创建带日期的新版本。脚本不会覆盖旧卡。除非用户明确要求，不删除旧卡。
