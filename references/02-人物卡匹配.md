# 人物卡匹配

## 读取范围

1. 按 `00-状态管理.md` 调用 `current --json` 和 `list --json`。
2. 使用返回的 `absolute_path` 读取当前卡和候选卡。
3. 候选范围包括 Skill 内的种子卡和外置状态目录中的专属卡。
4. 只匹配脚本实际列出的卡。`validate` 报告索引冲突时，不静默修复；需要修复时调用 `repair-index`。

## 优先级

- 用户本轮明确指定的卡：直接使用。
- 已确认的当前人物卡：同等适配时优先。
- 历史专属卡：优先于种子卡。
- 种子卡：没有合适专属卡时使用。

优先级只用于打破适配度接近的情况；明显不适配的当前卡不能压过明显更合适的候选。

## 评估维度

按以下顺序评估，每项只用 `✅ 高度匹配 / ⭕ 部分匹配 / ❌ 不匹配`，不要编造百分比：

1. **领域**：主赛道和细分议题是否吻合。
2. **受众**：受众认知、痛点和内容场景是否吻合。
3. **目标**：科普、说服、转化、共鸣等目标是否吻合。
4. **风格**：情感温度、专业度、口语度、攻击性是否吻合。
5. **结构**：原文逻辑能否自然映射到卡片常用结构。
6. **边界**：是否触碰用户禁忌，是否要求虚构身份或经历。

领域或边界为 `❌` 时，原则上不选为第一名。选出前三名后，给出一名首选和最多两名备选。

## 面向用户的简要输出

```markdown
推荐人物卡：{名称}
推荐理由：{领域、受众、风格、结构中最关键的 2–3 点}
需要调整：{为了贴合用户而弱化或加强的特征}
备选：{可选；名称与一句差异}
```

用户要求“匹配人物卡”而未要求立即改写时，推荐后等待确认。用户要求改写时，自动使用首选卡继续，不中断。
