# 状态管理

## 职责边界

使用模型完成画像提取、人物卡匹配、人物卡内容生成和文案改写。使用 `scripts/persona_state.py` 完成确定性状态操作：

- 初始化外置状态目录
- 列出种子卡和专属卡
- 读取或设置当前人物卡
- 安全保存专属卡且不覆盖旧文件
- 追加反馈记录
- 校验或修复人物卡索引

脚本只使用 Python 标准库，不调用模型，也不决定哪张人物卡更合适。

## 状态目录

按以下顺序确定：

1. 命令行 `--state-dir`
2. 环境变量 `PERSONA_WRITING_HOME`
3. 默认 `~/.persona-writing-skill`

种子卡始终从 Skill 的 `cards/seeds/` 读取。专属卡和可变记录写入状态目录，避免污染 Skill 的 Git 工作区或意外发布私人资料。

`--state-dir` 可以放在子命令之前或之后，两种写法等价。

## 常用命令

在 Skill 根目录执行。需要机器可读结果时始终追加 `--json`。

```bash
# 初始化
python3 scripts/persona_state.py init --json

# 列出全部卡片；每张卡均返回 absolute_path
python3 scripts/persona_state.py list --json

# 读取当前卡
python3 scripts/persona_state.py current --json

# 设置当前卡：参数可用名称、文件名或相对路径
python3 scripts/persona_state.py set-current "小五狼" \
  --basis "用户明确指定" --json

# 保存模型已生成并通过质量检查的 Markdown 卡片
python3 scripts/persona_state.py save \
  --name "专属人物卡名称" \
  --source "/临时目录/人物卡.md" \
  --domain "主要领域" \
  --style "核心风格" \
  --set-current \
  --basis "用户确认的专属人物卡" \
  --json

# 追加反馈
python3 scripts/persona_state.py add-feedback \
  --card "人物卡名称" \
  --type execution \
  --feedback "开头太长" \
  --result "压缩开头并重新优化" \
  --json

# 只检查，不自动修复
python3 scripts/persona_state.py validate --json

# 明确修复索引
python3 scripts/persona_state.py repair-index --json
```

`add-feedback --type` 接受 `execution`、`card`、`执行问题` 或 `人物卡问题`。

## 调用规则

- 调用 `list` 或 `current` 后，通过返回的 `absolute_path` 读取人物卡。
- 保存前先按 `04-人物卡模板.md` 和 `05-人物卡质量检查.md` 生成、检查卡片，再把卡片写入临时文件并调用 `save`。
- 不把整段 Markdown 放进 shell 参数；使用 `--source` 文件或 `--source -` 从标准输入读取。
- `validate` 返回非零状态表示发现错误；它不会改文件。只有用户要求或确认后才调用 `repair-index`。
- 脚本拒绝绝对卡片索引、父目录跳转和不安全文件名，并使用原子写入更新记录。
- 脚本不可用时，可继续使用种子卡完成改写；保存、当前卡和反馈功能必须明确标记为未持久化。
