# 人物卡质量检查

保存前逐项检查。任一硬性项失败时先修正，不保存半成品。

## 硬性检查

- [ ] 名称、定位、领域、受众、目标彼此一致。
- [ ] 所有真实身份、履历和偏好均有用户明示或样文证据；创作设定已明确标注。
- [ ] 不含敏感属性猜测，不把单篇文本表现当作用户永久身份。
- [ ] 至少包含 3 种标题、3 种开头、2 种主体结构和 2 种结尾。
- [ ] 表达特征足够具体，能指导句式、节奏、词汇和信息密度。
- [ ] 明确“不得虚构用户经历、不得冒充卡片人物、不得机械复制经典原句”。
- [ ] 与现有卡片至少存在一项清晰、可执行的差异。
- [ ] 文件名安全且不覆盖已有卡；索引和当前指针能够同步更新。

## 适配性检查

- 卡片是否能覆盖用户的主赛道，而非只适合一篇样文？
- 风格是否符合用户可长期维持的表达方式？
- 攻击性、夸张度、专业度和口语度是否有清晰边界？
- 推荐结构是否能服务用户目标，而非只追求“像某个人”？
- 是否把人物卡的原句改写成了可迁移的表达规律？

## 摘要输出

向用户展示：

```markdown
人物卡：{名称}
核心定位：{一句话}
最突出特征：{3 项}
适用内容：{2–4 项}
主动规避：{2–4 项}
```
