# 人物文风工厂（persona-writing-skill）

一个面向 Codex 的中文人物卡与文风改写 Skill。

它把“人物卡管理”和“文案改写”放进同一个闭环：可以根据用户画像推荐人物卡、生成并保存专属人物卡，也可以使用指定人物卡、当前人物卡或自动匹配的人物卡完成文案改写。

> 本 Skill 学习的是人物卡中的抽象表达规律，例如结构、节奏、句式、语气和修辞方式。它不会机械复制经典原句，也不会冒充卡片人物或虚构用户经历。

## 核心能力

- 根据用户画像、原文领域、受众和表达目标匹配人物卡
- 内置 8 张种子人物卡，并支持持续增加专属卡
- 保存当前人物卡，后续可直接使用“我的风格”改写
- 自动执行“立骨 → 铸魂 → 画皮 → 动嘴”四阶段流程
- 区分“本次执行问题”和“人物卡不匹配”，形成反馈闭环
- 保留原文事实、核心观点和必要限定，避免虚构数据、引语与经历
- 所有运行资源均为 Markdown，无 Python 运行依赖

## 工作模式

### 1. 匹配或创建人物卡

```text
根据我的画像匹配一个人物卡
给我推荐一种适合科技科普的表达风格
这些人物卡都不像我，帮我生成一张专属卡
保存这张人物卡并设为当前卡
```

流程：

```text
提取创作画像
→ 动态读取种子卡和专属卡
→ 推荐最匹配的卡
→ 不满意时生成专属卡
→ 保存并设置为当前卡
```

### 2. 指定人物卡改写

```text
用小五狼的人物卡改写下面这段
按照杰明科技官的表达方式重写
```

用户指定人物卡时会跳过自动匹配，直接执行：

```text
立骨 → 画皮 → 动嘴
```

### 3. 使用当前人物卡

```text
把下面内容改成我的风格
继续用当前人物卡改写
```

Skill 会读取 `records/当前人物卡.md` 指向的卡片，不需要每次重新选择。

### 4. 自动选择人物卡

```text
帮我把这段改得更适合短视频口播
```

没有指定人物卡且没有当前卡时，Skill 会动态读取全部实际存在的卡片：

```text
立骨 → 铸魂自动匹配 → 画皮 → 动嘴
```

### 5. 处理不满意反馈

```text
开头太夸张了
整体不像我，受众也不对
```

- 局部执行问题：保留当前卡，重新执行画皮和动嘴。
- 人物卡问题：生成专属卡、保存、设为当前卡，并立即重写上一版原文。

## 四阶段改写流程

| 阶段 | 作用 |
|---|---|
| 立骨 | 从 12 个维度拆解原文的观点、受众、事实、结构、语言和传播意图 |
| 铸魂 | 从实际存在的卡片中按领域、受众、目标、风格、结构和边界动态匹配 |
| 画皮 | 迁移人物卡的抽象结构、节奏和语言规律，生成完整初稿 |
| 动嘴 | 检查内容、事实、身份、节奏和字数，输出最终版本 |

当用户指定人物卡或已有有效当前卡时，会跳过“铸魂”。

## 安装

将仓库克隆到 Codex Skills 目录：

```bash
git clone https://github.com/YiHarvest/persona-writing-skill.git ~/.codex/skills/persona-writing-skill
```

重新启动或刷新 Codex 会话后，可以显式调用：

```text
使用 $persona-writing-skill 为我的内容匹配人物卡并完成文风改写。
```

也可以直接使用自然语言提出人物卡匹配、创建或文案改写请求。

## 新增专属人物卡

专属卡保存在 `cards/custom/`，文件名遵循：

```text
YYYYMMDD-人物卡名称.md
```

新增后需要同步：

1. 在 `records/人物卡索引.md` 中增加索引。
2. 若设为当前卡，更新 `records/当前人物卡.md`。
3. 将可长期复用的风格反馈写入 `records/用户反馈记录.md`。

专属卡结构和保存规则分别见：

- `references/04-人物卡模板.md`
- `references/05-人物卡质量检查.md`
- `references/03-人物卡生成.md`

## 内容与事实边界

- 不虚构原文没有的事实、数据、来源、引语或个人经历。
- 不把人物卡中的履历、家庭和第一人称经历移植给用户。
- 不机械复制人物卡中的经典金句或高度可识别原句。
- 不把卡片人物的署名或关注口播默认带入成稿。
- 对新闻、政策、价格、职位、科技进展及医学、法律、金融等时效或高风险信息，先核验再补充。
- 原文信息无法核验时，只改写用户已经提供的内容，并保留必要的不确定性。

## 验证

项目遵循 Codex Skill 目录和 frontmatter 规范，可使用官方校验脚本检查：

```bash
python /path/to/skill-creator/scripts/quick_validate.py .
```

Skill 运行本身不依赖 Python；Python 仅用于可选的开发期结构校验。
